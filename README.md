# Хрестики-Нулики (ООП)

Консольна реалізація класичної гри "Хрестики-Нулики", розроблена на мові C# з використанням принципів об'єктно-орієнтованого програмування та багатошарової архітектури. Проект включає систему користувачів, облік рейтингу та інтелектуальних ботів різної складності.

## Архітектура проекту

Проект розділений на логічні рівні (Layers) для забезпечення чистоти коду та легкості підтримки:

- **Core (Основна логіка)**: Містить ігрові сутності (Board, GameEngine), абстракції гравців та логіку ігрових механік.  
- **Data (Робота з даними)**: Імітація бази даних за допомогою GameDbContext та UserRepository. Дані зберігаються у форматі JSON.  
- **Services (Сервіси)**: Шар бізнес-логіки, що відповідає за реєстрацію, автентифікацію та безпеку сесії користувача.  
- **UI (Інтерфейс користувача)**: Консольна комунікація, інтерактивне ігрове поле з підтримкою керування стрілками клавіатури.  

## Ключові особливості

- **Система акаунтів**: Реєстрація, вхід та можливість повного видалення профілю.  
- **Рейтингова система**: Автоматичне нарахування або зняття балів рейтингу за результатами гри (Перемога: +25, Поразка: -25, Нічия: +5).  
- **Історія ігор**: Детальний лог усіх зіграних партій із зазначенням дати, опонента та результату.  
- **Рівні складності ботів**:  
  - Easy: Випадкові ходи.  
  - Medium: Пріоритет на блокування ходів гравця та власну перемогу.  
  - Hard: Використання алгоритму Minimax для непереможної стратегії.  
- **Гнучкість налаштувань**: Можливість вибору сторони гри (X або O) та рівня складності перед початком матчу.  

## Технологічний стек

- Мова: C# (.NET Core / .NET 5+)  
- Формат даних: JSON (System.Text.Json)  
- Принципи: ООП (Інкапсуляція, Наслідування, Поліморфізм), Repository Pattern, Strategy Pattern  

## Як запустити

1. Склонуйте репозиторій.  
2. Відкрийте файл рішення `.sln` у Visual Studio або JetBrains Rider.  
3. Переконайтеся, що консоль підтримує кодування UTF-8 для коректного відображення псевдографіки поля.  
4. Запустіть проект.  

## Демонстрація ООП в проекті

- **Поліморфізм**: Різні стратегії ботів реалізують інтерфейс `IBotStrategy`.  
- **Наслідування**: Класи `UserPlayer` та `BotPlayer` успадковуються від базового абстрактного класу `Player`.  
- **Інкапсуляція**: Логіка перевірки виграшних комбінацій та стану поля прихована всередині класу `Board`.
